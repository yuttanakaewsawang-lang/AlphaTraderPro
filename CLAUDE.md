# Apollo Auto Trade — Project Rules

## Stack
- **Frontend:** React + TypeScript + Vite + TailwindCSS → `frontend/dist/`
- **Backend:** FastAPI (port 8000) + MetaTrader5 Python + SQLite
- **Desktop:** PyWebView 6.2.1 wrapping `http://127.0.0.1:8000`
- **Exe:** PyInstaller via `run_app.spec` → `dist/ApolloAutoTrade.exe`
- **Data dir (exe):** `%APPDATA%\ApolloAutoTrade\` — app.db, license.json, logs/ (ไม่มี dist/ cache แยกแล้ว — v1.0.85+ เสิร์ฟ frontend ตรงจากในตัว exe เสมอ, single-file update)

## Key Files

|------|------|

## Release Commands
```powershell
.\release.ps1 -Notes "..."
```
- **Single-file only — ไม่มี `-Patch` mode แล้ว** (v1.0.85+) — build exe เต็มทุกครั้งไม่ว่าจะแก้ frontend หรือ backend
  ก็ตาม เพราะ `-Patch` เดิม (แตก zip ทับ `%APPDATA%\ApolloAutoTrade\dist`) ทำให้เวลา copy exe ไปลง VPS มือ
  (ไม่ผ่าน in-app updater) frontend เก่าค้างอยู่ใน cache ไม่ยอมอัปเดต — ตอนนี้ copy exe ไฟล์เดียวทับตัวเดิมจบ ชัวร์ 100%
- Auto-increments PATCH version, builds, pushes GitHub Release
- **Close exe first** — running exe locks dist/ApolloAutoTrade.exe → PermissionError
- **อัปเดตบน VPS: ปิด exe เดิมให้หมดก่อน copy ทับเสมอ** — ถ้า copy ทับตอน exe เก่ายังรันอยู่ โปรเซสในหน่วยความจำจะยังเป็นโค้ดเก่าจนกว่าจะปิด-เปิดใหม่

## Dev Preview (ทดลอง UI/logic โดยไม่แตะ DB หลัก)
```powershell
.\dev.ps1                 # build frontend + เปิดหน้าต่างแอปเต็ม (instance: uitest — DB แยก)
.\dev.ps1 -SkipBuild      # เปิดซ้ำเร็วๆ ไม่ build ใหม่ (frontend ไม่ได้แก้)
.\dev.ps1 -Mode hot       # uvicorn --reload + vite dev → http://localhost:5173 แก้ UI เห็นทันที
```
- **บังคับ `--instance` เสมอ** — DB แยกที่ `instances\<name>\` ไม่มีทางชน DB dev หลัก/auto-resume เปิดไม้จริง
- **ห้ามรัน `python run_app.py` เปล่าๆ เพื่อทดลอง UI** — ใช้ DB หลักซึ่งอาจมี resume state ค้าง → บอทเทรดทันทีที่ login
- `start.bat` เดิม (uvicorn+vite ไม่มี instance) ถือว่า deprecated สำหรับงานทดลอง — ใช้ `dev.ps1 -Mode hot` แทน

## Config — 4 sources must stay in sync (SMC)
When changing any strategy default, update **all four**:
1. `Strategy.py` → `SMCStrategy.__init__`
2. `database.py` → `init_db()` CREATE TABLE defaults
3. `backtest.py` → `DEFAULT_CONFIG`
4. `frontend/src/components/StrategyView.tsx` → `RECOMMENDED_DEFAULTS`

Missing one = live/UI/reset drift (caused bugs before).

## Config — 3 sources must stay in sync (Sniper)
Sniper มี frontend config UI แยกยังไม่ทำ (deferred) — sync แค่ 3 จุด:
1. `SniperStrategy.py` → `SniperStrategy.__init__`
2. `database.py` → `sniper_config` CREATE TABLE defaults
3. `sniper_backtest.py` → `DEFAULT_CONFIG`

Signal/SL/TP formula (N-bar breakout, measured-move TP) ใน `SniperStrategy.execute_logic()`
ต้องตรงกับ `sniper_backtest.py` เป๊ะ เหมือนที่ `Strategy.py`/`backtest.py` sync กันสำหรับ SMC.

## Config — 3 sources must stay in sync (Swing / Reversal / Grid)
กลยุทธ์ EA3-5 ใช้กติกาเดียวกับ Sniper — sync 3 จุดต่อกลยุทธ์:
1. `<Engine>Strategy.py` → `__init__` (Swing/Reversal/Grid)
2. `database.py` → `swing_config` / `reversal_config` / `grid_config` CREATE TABLE defaults
3. `<engine>_backtest.py` → `DEFAULT_CONFIG`
(+ frontend `EngineConfigPanel.tsx` → `ENGINE_DEFS[engine].recommended`)

เงื่อนไขสัญญาณของ Swing/Reversal **แชร์ฟังก์ชันเดียวกันระหว่าง live กับ backtest จริงๆ**
(`swing_backtest.swing_signal`, `reversal_backtest.reversal_signal`/`rsi_series` ถูก import
เข้า live class ตรงๆ) — แก้สูตรที่ฟังก์ชันเดียวจบ ไม่มีทาง drift; ส่วน SL/TP live anchor
กับ tick จริง ณ ตอนเข้า (convention เดียวกับ Sniper) และ Grid basket semantics ใน
`GridStrategy` ต้อง mirror `grid_backtest.py` (pessimistic: เติมกริด → เช็ค stop → เช็ค TP)

## MT5 Terminal (VPS Rule)
- `initialize_mt5()` **requires** `terminal_path` — raises RuntimeError if missing
- After `mt5.initialize(path=...)`, verify `terminal_info().path` matches expected dir
- If mismatch → `mt5.shutdown()` + raise error → never silently connect to wrong terminal

## Candle Index Convention
```python
df.iloc[-1]  # forming candle — NEVER use for signals
df.iloc[-2]  # row1: last closed bar (use for entry signal)
df.iloc[-3]  # row2: bar before that (use for engulfing check)
```

## Common Pitfalls
- `encoding="ascii"` in bat files → never put Thai text → UnicodeEncodeError (fixed v1.0.28)
- `display:none` chart container → chart loses dimensions on remount → use absolute overlay instead
- `mt5.initialize()` without path on VPS with 2 terminals → connects to wrong one
- Order ซ้ำแท่งเดียว → ต้อง lock `_last_entry_bar_time` จาก `row1['time']`
- 4 config sources ไม่ sync → "Reset to Recommended" เขียนค่าเก่ากลับ DB
